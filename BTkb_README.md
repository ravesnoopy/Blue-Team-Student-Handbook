# SOC Handbook

A practical cybersecurity knowledge base for Security Operations Center professionals. Focused on incident response, threat detection, Windows internals, and defensive security operations.

---

## 📋 Structure

This handbook is organized by core security domains:

- **Incident Response** — NIST framework, containment, recovery, root cause analysis
- **Windows Internals** — LSASS, registry, services, authentication mechanisms
- **Active Directory** — Kerberos, NTLM, GPO, lateral movement detection
- **Networking** — DNS, DHCP, TCP/UDP, TLS, packet analysis
- **Threat Detection** — Detection engineering, SIGMA rules, MITRE ATT&CK mapping
- **DFIR** — Memory analysis, timeline analysis, artifacts, evidence handling
- **SIEM** — Log analysis, correlation, alerting (Splunk, Elastic)
- **Malware Analysis** — Persistence, evasion, behavioral analysis
- **Detection Engineering** — Building detections from threat intelligence
- **Linux** — Hardening, log analysis, post-exploitation artifacts
- **Cloud Security** — AWS, Azure security posture
- **MITRE ATT&CK** — Tactics and techniques reference
- **Cheat Sheets** — Quick reference guides
- **Labs & Notes** — Real investigation case studies
- **Interview Prep** — Common SOC interview questions

---

## 🎯 Purpose

This is not a textbook. Each document is:
- **Practical** — Written from hands-on experience
- **Concise** — Never exceeds 2 pages
- **Connected** — Topics link to related concepts
- **Searchable** — Clear structure for quick reference during incidents

---

## 📖 How to Use

1. **Use as reference during investigations** — Quick lookup for detection strategies, Windows artifacts, network protocols
2. **Prepare for interviews** — Understand concepts deeply, not just memorize definitions
3. **Build detection logic** — Study MITRE mappings and detection examples
4. **Learn through connections** — Follow related topics to build mental models
